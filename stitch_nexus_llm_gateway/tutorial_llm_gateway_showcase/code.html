Module 02

Routing Configuration

Current Task

Define model fallbacks and load balancing strategies. If a primary provider (e.g., GPT-4) hits a rate limit or latency threshold, the gateway automatically redirects to your defined secondary model.

ROUTING.JSON data_object 
{
  "strategy": "lowest_latency",
  "targets": [
    { "id": "model_a", "weight": 80, "fallback": "model_b" },
    { "id": "model_b", "weight": 20 }
  ],
  "retries": 3,
  "timeout": "450ms"
}
